# TodoList V2.0 개발 로드맵 (Development Roadmap)

사용자님이 제안하신 7가지 핵심 기능을 바탕으로, 안정성을 최우선으로 한 개발 계획입니다.
기존 기능이 망가지지 않도록 **'단계별 개발'**과 **'안전 장치(Git Branching)'** 전략을 사용합니다.

## 🛡️ 안전 장치 및 개발 전략
1.  **Git 브랜치 전략**:
    -   `main`: 언제나 배포 가능한 안정적인 버전 (v1.x 최종본 유지)
    -   `develop`: V2.0 기능들이 통합되는 브랜치
    -   `feature/*`: 각 기능별 작업 브랜치 (예: `feature/kanban`, `feature/calendar-sync`)
2.  **백업**:
    -   주요 데이터 구조 변경 전 Firestore 데이터 백업 스크립트 실행.

---

## 📅 단계별 실행 계획 (Phased Execution Plan)

### 1단계: UI/UX 혁신 (시각적 변화, 리스크 낮음)
기존 데이터 구조를 크게 건드리지 않으면서 사용자가 가장 먼저 체감할 수 있는 기능들입니다.

-   **3. 이번 달 할일 UI 개선 (TickTick 스타일)**
    -   현재 월간 뷰: 단순 달력 형태.
    -   변경 목표: 달력 아래에 해당 날짜의 할일 목록이 리스트 형태로 바로 뜨는 직관적인 구조.
-   **4. 작업 그룹화 & 칸반 보드 (Kanban Board)**
    -   '진행 중 / 대기 / 완료' 또는 '프로젝트 별'로 카드를 드래그 앤 드롭하는 보드 뷰 추가.
    -   기존 `status`나 `project` 필드 활용.
-   **5. 외관 테마 확장 (Fun Themes)**
    -   기존 다크/라이트 모드를 넘어, '사이버펑크', '자연', '파스텔' 등 다양한 색상 팔레트 적용.

### 2단계: 네이티브 기능 확장 (안드로이드 기능, 중급 난이도)
Capacitor와 안드로이드 네이티브 코드를 다루는 단계입니다.

-   **7. 모바일 알림 다각화**
    -   알림 사운드 변경, '완료'/'미루기' 버튼이 있는 알림(Action Buttons).
    -   중요도(긴급)에 따라 알람 소리 구분.
-   **2. 위젯 기능 (Android Widget)**
    -   홈 화면에 '오늘의 할일' 리스트 위젯 추가.
    -   Native Android (Kotlin/Java) 개발 필요.

### 3단계: 연동과 협업 (구조적 변화, 리스크 높음)
데이터베이스 구조 변경과 외부 API 연동이 필요하므로 가장 마지막에 신중하게 진행합니다.

-   **6. 구글 캘린더 연동**
    -   구글 계정 권한 추가 필요.
    -   양방향 동기화(앱에서 추가하면 구글 캘린더에도 추가) 구현.
-   **1. 일정 공유 (협업 기능)**
    -   가장 큰 작업. `Todo` 데이터에 `sharedWith` 필드 추가.
    -   초대 링크 생성, 권한 관리(읽기 전용/수정 가능), 실시간 동기화 확인.

## ✅ 상세 구현 계획 (Implementation Details)

### Feature 3: TickTick 스타일 월간 뷰
-   **목표**: 달력의 날짜를 누르면 화면 전환 없이 하단(또는 오버레이)에 리스트가 부드럽게 펼쳐짐.
-   **작업**: `MonthlyCalendarView.tsx` 리팩토링, Framer Motion 애니메이션 적용.

### Feature 4: 칸반 보드
-   **목표**: `dnd-kit` 라이브러리를 사용하여 부드러운 드래그 앤 드롭 구현.
-   **구조**: `BoardView.tsx` 신규 생성. `status` 별 컬럼핑.

### Feature 2: 안드로이드 위젯
-   **목표**: 앱을 켜지 않고도 홈 화면에서 할일 확인.
-   **기술**: Capacitor 커스텀 플러그인 또는 Android Native Code (`AppWidgetProvider`) 작성.

### Feature 1: 일정 공유
-   **DB 변경**:
    ```typescript
    interface Todo {
      ...
      ownerId: string; // 생성자
      sharedWith: string[]; // 공유받은 사용자 UID 목록
    }
    ```
-   **UI**: '공유' 버튼 추가, 이메일로 사용자 초대 모달.

## 🚀 시작 제안
가장 먼저 **'3. 이번 달 할일 UI (TickTick 스타일)'**과 **'4. 칸반 보드'**부터 시작하는 것을 추천합니다.
눈에 보이는 변화가 크고, 기존 로직을 망가뜨릴 위험이 가장 적기 때문입니다.
